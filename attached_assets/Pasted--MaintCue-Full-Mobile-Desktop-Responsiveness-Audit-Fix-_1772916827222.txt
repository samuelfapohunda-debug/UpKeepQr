## 🛠️ **HOW TO WORK**

1. Go through each surface **one at a time** in priority order
2. For each surface: **identify all issues first**, then fix them all before moving on
3. Use **Tailwind responsive prefixes** (`sm:`, `md:`, `lg:`, `xl:`) consistently
4. **Mobile-first** approach: base styles for mobile, add `sm:` and up for larger screens
5. Desktop fixes use `lg:` and `xl:` prefixes
6. **Do NOT change:**
   - Functionality, logic, or data fetching
   - Business rules or validation
   - Color tokens or design system values
   - API endpoints or route structure
7. **Only fix:** Layout, spacing, sizing, breakpoints, overflow

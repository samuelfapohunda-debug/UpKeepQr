# 🎯 **Replit Agent Instructions: Fix Subscription Webhook 404 Error**

---

## **OBJECTIVE**
Fix the 404 error on `/api/stripe/subscription-webhook` endpoint by correcting the import path in `server/src/routes/index.ts`. The webhook handler function is being imported from the wrong file.

---

## **PROBLEM IDENTIFIED**

**Current state (BROKEN):**
```typescript
// server/src/routes/index.ts - Line 22
import { registerSubscriptionRoutes, registerSubscriptionWebhookHandler } from '../../routes/subscription.js';
```

**Issue:** `registerSubscriptionWebhookHandler` does NOT exist in `subscription.js` - it's in `webhooks.ts`. This causes the import to fail silently and the webhook route is never registered, resulting in 404 errors.

**Evidence:**
```bash
curl -I https://maintcue.com/api/stripe/subscription-webhook
# Returns: HTTP/2 404 ❌
```

---

## **THE FIX**

Update `server/src/routes/index.ts` to import from the correct files:

### **Step 1: Locate the broken import**

Find this line in `server/src/routes/index.ts` (around line 22):

```typescript
import { registerSubscriptionRoutes, registerSubscriptionWebhookHandler } from '../../routes/subscription.js';
```

### **Step 2: Replace with correct imports**

Change it to:

```typescript
import { registerSubscriptionRoutes } from '../../routes/subscription.js';
import { registerSubscriptionWebhookHandler } from '../../routes/webhooks.js';
```

**Explanation:**
- `registerSubscriptionRoutes` is correctly in `subscription.js` ✅
- `registerSubscriptionWebhookHandler` is actually in `webhooks.ts` ✅

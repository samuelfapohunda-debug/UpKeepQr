## ğŸ¨ STEP 2: CREATE MARQUEE COMPONENT

### Option A: Create New Component (Recommended)

**File:** `client/src/components/ScrollingAnnouncement.tsx`

```tsx
import React from 'react';
import styles from './ScrollingAnnouncement.module.css';

interface AnnouncementItem {
  id: string;
  text: string;
}

interface ScrollingAnnouncementProps {
  items?: AnnouncementItem[];
  speed?: number; // Duration in seconds for one complete scroll
}

const ScrollingAnnouncement: React.FC<ScrollingAnnouncementProps> = ({ 
  items = [
    { id: '1', text: 'Trusted by 100+ Realtor subscriptions in North America' },
    { id: '2', text: '5,000+ homeowners trust MaintCue for maintenance tracking' },
    { id: '3', text: 'Never miss a maintenance task again with smart reminders' }
  ],
  speed = 30 
}) => {
  // Duplicate items for seamless infinite scroll
  // Use useMemo to prevent re-creation on every render
  const duplicatedItems = React.useMemo(() => [...items, ...items], [items]);

  return (
    <div className={styles.marqueeContainer}>
      <div 
        className={styles.marqueeContent}
        style={{ animationDuration: `${speed}s` }}
      >
        {duplicatedItems.map((item, index) => (
          <span key={`${item.id}-${index}`} className={styles.marqueeItem}>
            {item.text}
            <span className={styles.separator}>â€¢</span>
          </span>
        ))}
      </div>
    </div>
  );
};

export default ScrollingAnnouncement;
```

## ğŸ¯ STEP 5: CONFIGURE ANNOUNCEMENTS

### Strategic Announcement Ordering (UX Psychology)

**Persuasion Architecture Principle:**
Order announcements from **strongest proof â†’ feature â†’ benefit** to maximize trust and conversion.

**Recommended Sequence:**
```tsx
<ScrollingAnnouncement 
  items={[
    // 1. STRONGEST TRUST SIGNAL (social proof with numbers)
    { id: '1', text: 'Trusted by 100+ Realtor subscriptions in North America' },
    
    // 2. SECONDARY PROOF (user volume)
    { id: '2', text: '5,000+ homeowners never miss maintenance tasks' },
    
    // 3. KEY FEATURE (what makes it unique)
    { id: '3', text: 'Smart QR codes track your home\'s entire service history' },
    
    // 4. BENEFIT (outcome for user)
    { id: '4', text: 'Climate-aware reminders for seasonal maintenance' },
    
    // 5. LOW-BARRIER CTA (reduces friction)
    { id: '5', text: 'Setup in under 2 minutes - No credit card required' }
  ]}
  speed={40}
/>
```

**Why This Order Works:**
1. **Trust first** - Establishes credibility immediately
2. **Proof of scale** - Shows product is validated by others
3. **Unique value** - Explains what makes you different
4. **Benefit realization** - Shows tangible outcomes
5. **Removes objections** - Addresses barriers to signup

**Anti-Pattern (Avoid):**
```tsx
// âŒ Random order - no persuasion strategy
items={[
  'Setup in 2 minutes',           // CTA before trust
  'Climate-aware reminders',      // Feature before proof
  'Trusted by 100+ Realtors'      // Trust buried at end
]}
```

## ğŸ” TROUBLESHOOTING

### Issue: Animation is jerky/laggy
**Solution:**
- Add `will-change: transform` to `.marqueeContent`
- Use `transform: translateX()` instead of `left`
- Reduce number of announcement items
- Increase animation duration (slower = smoother)

### Issue: Visible gap when loop restarts
**Solution:**
- Ensure items are duplicated: `[...items, ...items]`
- Animation should go from `0%` to `-50%` (not `-100%`)
- Check that `display: inline-flex` is on `.marqueeContent`

### Issue: Text is unreadable (too fast)
**Solution:**
- Increase `speed` prop (30-40 seconds recommended)
- Reduce font size on mobile
- Add longer padding between items

### Issue: Not responsive on mobile
**Solution:**
- Add media queries for font size and padding
- Test on actual devices, not just DevTools
- Reduce fade gradient width on small screens
